# 第4週目 課題4 レポート（シフト加算法）

## 課題内容
DC命令に与えられる2値について、その積の下位16ビットを求め、DS命令で定義されるメモリ領域に格納するプログラムを作成せよ。
シミュレーション実験によって #005F × #007D を計算し、計算にかかる時間、実行する命令数、CPI値を示せ。

---

## アルゴリズム：シフト加算法

乗数（GR2）の最下位ビットを毎回検査し、1であれば被乗数（GR1）を結果（GR0）に加算する。
その後 GR1 を1ビット左シフト（×2）、GR2 を1ビット右シフト（÷2）し、GR2 が0になるまで繰り返す。

```
手順:
1. GR0を0で初期化する
2. GR3 = GR2 AND 1（最下位ビットを取り出す）
3. GR3が0でなければ GR0 += GR1
4. GR1 を左へ1ビットシフト（SLL GR1,1）
5. GR2 を右へ1ビットシフト（SRL GR2,1）
6. GR2が0でなければ2へ移動する
7. 呼び出し元へ戻る
```

### 足し算繰り返し法との比較
- 足し算繰り返し法: ループ回数 = 乗数の値（125回）
- シフト加算法:     ループ回数 = 乗数のビット数（7回）
- #007D = 0111 1101（2進数） → 7ビット → 7回のループで完了

## CASL IIプログラム

```
MAIN2    START
         LD      GR1, DATA1   ; GR1 = #005F（被乗数）
         LD      GR2, DATA2   ; GR2 = #007D（乗数）
         CALL    MULT2        ; 乗算副プログラム呼び出し
         ST      GR0, ANS     ; 結果をメモリに格納
         RET
DATA1    DC      #005F        ; 被乗数
DATA2    DC      #007D        ; 乗数
ANS      DS      1            ; 結果格納領域
         END

MULT2    START
         PUSH    0, GR1       ; GR1をスタックに退避
         PUSH    0, GR2       ; GR2をスタックに退避
         PUSH    0, GR3       ; GR3をスタックに退避
         LAD     GR0, 0       ; GR0 = 0（結果初期化）
LOOP     LD      GR3, GR2     ; GR3 = GR2（コピー）
         AND     GR3, ONE     ; GR3 = GR2 AND 1（最下位ビット）
         JZE     SHIFT        ; GR3=0（偶数）ならADDAをスキップ
         ADDA    GR0, GR1     ; GR0 += GR1（奇数のとき加算）
SHIFT    SLL     GR1, 1       ; GR1 を左1ビットシフト
         SRL     GR2, 1       ; GR2 を右1ビットシフト
         JNZ     LOOP         ; GR2≠0ならループ継続
         POP     GR3          ; GR3を復元
         POP     GR2          ; GR2を復元
         POP     GR1          ; GR1を復元
         RET                  ; 呼び出し元へ戻る
ONE      DC      1
         END
```


======================

MAIN2    START
         LD      GR1, DATA1  
         LD      GR2, DATA2 
         CALL    MULT2     
         ST      GR0, ANS   
         RET
DATA1    DC      #005F     
DATA2    DC      #007D   
ANS      DS      1        
         END

MULT2    START
         PUSH    0, GR1     
         PUSH    0, GR2    
         PUSH    0, GR3   
         LAD     GR0, 0      
LOOP     LD      GR3, GR2  
         AND     GR3, ONE    
         JZE     SHIFT      
         ADDA    GR0, GR1   
SHIFT    SLL     GR1, 1     
         SRL     GR2, 1     
         JNZ     LOOP      
         POP     GR3      
         POP     GR2       
         POP     GR1       
         RET              
ONE      DC      1
         END


### 工夫した点
- CALL命令で副プログラムとして実装し、メインプログラムと分離した
- PUSH/POP命令でGR1,GR2,GR3の元の値を保護し、呼び出し元に影響を与えない設計にした
- GR2を直接ANDせず GR3 にコピーしてANDすることで、GR2のビット列を破壊しない
- JZE でビット=0のとき加算をスキップし、無駄な演算を省いた

## シミュレーション結果

### 実験環境
- クロック周波数: 125MHz（周期8ns）
- タイムスケール: 1ns/1ps
- START立ち上がり: 122ns

### 計算結果
```
memory[11] = #2E63
```
検算: #005F × #007D = 95 × 125 = 11875 = #2E63 ✓

### 計算時間
```
DONE=1になった時刻: 4,692ns
計算時間 = 4,692ns - 122ns = 4,570ns
```

### 命令数

ループ動作の詳細（#007D = 0111 1101、7回ループ）：

| 反復 | GR2（2進数） | 最下位ビット | ADDA実行 |
|------|-------------|-------------|---------|
| 1 | 0111 1101 | 1 | あり |
| 2 | 0011 1110 | 0 | なし |
| 3 | 0001 1111 | 1 | あり |
| 4 | 0000 1111 | 1 | あり |
| 5 | 0000 0111 | 1 | あり |
| 6 | 0000 0011 | 1 | あり |
| 7 | 0000 0001 | 1 | あり |

| 区間 | 命令 | 命令数 |
|------|------|--------|
| MAIN2 | LD, LD, CALL, ST, RET | 5 |
| MULT2 初期化 | PUSH, PUSH, PUSH, LAD | 4 |
| ループ（bit=1、6回） | LD + AND + JZE + ADDA + SLL + SRL + JNZ | 42 |
| ループ（bit=0、1回） | LD + AND + JZE + SLL + SRL + JNZ | 6 |
| MULT2 後処理 | POP, POP, POP, RET | 4 |
| **合計** | | **61** |

ループ回数が7回になる理由: DATA2 = #007D = 7ビット（0111 1101）であり、SRL命令で毎回1ビット右シフトして0になるまでループするため。

### CPI
```
総クロック数 = 4,570ns ÷ 8ns = 571クロック
命令数 = 61命令
CPI = 571 ÷ 61 ≈ 9.36
```

## まとめ

| 項目 | シフト加算法 | 足し算繰り返し法 |
|------|------------|----------------|
| 計算結果 | #2E63（=11875） | #2E63（=11875） |
| 計算時間 | 4,570ns | 28,858ns |
| 命令数 | 61命令 | 386命令 |
| CPI | 9.36 | 9.35 |

シフト加算法は命令数が386→61命令と大幅に削減され、計算時間も大きく短縮される。
乗数のビット数（7回）だけループすれば済むため、乗数の値（125回）に比例してループする足し算繰り返し法より効率的なアルゴリズムである。
