教師あり機械学習プロジェクト課題
1. 課題の目的
配布された訓練データを用いて二値分類モデルを構築し、未知のデータに対する予測性能を競う。
本課題では、単に訓練データに対する性能を高くするだけでなく、検証データまたは交差検証を用いて、未
知のデータに対して高い性能を持つモデルを構築することを目的とする。
データセットによって性質が異なるため、すべてのデータセットで同じ方法が有効とは限らない。授業で
扱った複数の手法を比較し、自分のデータに適した方法を選択すること。

2. 配布ファイル
次のファイルを配布する。
train.csv ：モデルの学習に使用する訓練データ
test_sample.csv ：プログラムの動作確認に使用するサンプルデータ
student_template.py ：提出プログラムのテンプレート
development.ipynb ：Google Colab上でプログラムを作成・確認するためのノートブック
本番評価用のテストデータは配布しない。採点時には、教員側で用意した非公開のテストデータを使用す
る。

3. データ形式
train.csv には、識別番号 id 、特徴量、正解ラベル y が含まれる。
id,x01,x02,...,x20,y
test_sample.csv には正解ラベル y が含まれない。
id,x01,x02,...,x20
本番のテストデータも test_sample.csv と同じ列構成である。ただし、行数および特徴量の値は異なる。

4. 実装する関数
提出するPythonファイルには、必ず次の関数を定義すること。
def train_and_predict(train_path: str, test_path: str):
...
この関数は、次の処理を行う。
train_path で指定された訓練データを読み込む。
分類モデルを学習する。
test_path で指定されたテストデータを読み込む。
テストデータの各行に対する予測ラベルを返す。
戻り値は、0または1からなる一次元配列とする。戻り値の要素数は、テストデータの行数と一致させるこ
と。
5. 使用できるライブラリ
次のライブラリを使用できる。
NumPy
pandas
SciPy
scikit-learn
これら以外のライブラリを使用してはならない。
scikit-learnに含まれる以下のようなモデルは使用できる。
ロジスティック回帰
線形判別分析
Naive Bayes
SVM
決定木
ランダムフォレスト
Gradient Boosting
正則化を用いた線形モデル
深層学習、外部データ、事前学習済みモデル、AutoML、外部APIは使用してはならない。
6. モデルの作成
モデルの作成方法は自由である。ただし、次の点を考慮すること。
訓練データと検証データを適切に分ける。
必要に応じて交差検証を用いる。
複数のモデルやハイパーパラメータを比較する。
必要に応じて標準化や正則化を用いる。
訓練データだけに過度に適合しないようにする。
採点時には、提出された最終プログラムだけを実行する。検証やモデル比較は、提出前に各自で行うこと。
7. プログラムの制約
提出プログラムは、次の条件を満たす必要がある。
•
関数名を train_and_predict とする。
2
引数を train_path と test_path の2つとする。
戻り値を0または1からなる一次元配列とする。
実行時にキーボード入力を要求しない。
Google Drive固有のパスを使用しない。
ファイル選択などの手動操作を要求しない。
インターネットに接続しない。
外部データを読み込まない。
train_path と test_path 以外のCSVファイルに依存しない。
テストデータの行数や値をプログラム内に直接記述しない。
実行時間の上限を超えない。
乱数を使用する場合は、結果を再現できるように random_state を指定すること。
8. 実行時間
採点時の実行時間には上限を設ける。
大規模なグリッドサーチ、極端に多い交差検証、非常に多数のモデルを用いたアンサンブルなど、実行時間
の長い処理は避けること。
採点時に制限時間を超えたプログラムは、正常に実行できなかったものとして扱う場合がある。
9. 動作確認
development.ipynb をGoogle Colabで開き、次のファイルをアップロードして動作確認すること。
train.csv
test_sample.csv
作成したPythonファイル

動作確認では、以下を確認すること。
エラーなく実行できる。
予測数が test_sample.csv の行数と一致する。
予測値が0または1である。
予測結果が一次元配列である。
test_sample.csv は動作確認用であり、その予測性能は評価しない。

10. 提出方法
MoodleにPythonファイルを1つ提出すること。
ファイル名は、必ず次の形式とする。
学生証番号.py
例として、学生証番号が 24RB1001 の場合は、次の名前で提出する。
3
24RB1001.py
次のようなファイル名は認めない。
kadai.py
24RB1001_final.py
24RB1001.ipynb
24RB1001.zip
提出するのはPythonファイルのみである。 prediction.csv 、Google Colabノートブック、訓練データなど
は提出しない。
11. 採点方法
教員側で、提出されたすべてのPythonファイルを一括実行する。
各学生の train_and_predict 関数に、割り当てられた訓練データと非公開のテストデータのパスを渡す。返
された予測結果を正解ラベルと比較して評価する。
予測性能の評価には balanced accuracy を用いる。
[
\frac{1}{2}
\left(
\text{クラス0の正解率}
•
\text{クラス1の正解率}
\right)
]
評価対象には、次の項目を含む。
指定されたファイル名で提出されている。
train_and_predict 関数が正しく定義されている。
プログラムがエラーなく実行できる。
戻り値の形式が正しい。
制限時間内に処理が終了する。
非公開テストデータに対する予測性能が高い。
12. 禁止事項
以下の行為を禁止する。
他者のプログラムをコピーする。
自分のプログラムを他者に提供する。
外部データを使用する。
非公開テストデータや正解ラベルの入手を試みる。
学生証番号に対応しないデータセットを使用する。
4
採点環境のファイルやシステムを操作するコードを書く。
ネットワーク通信を行うコードを書く。
提出プログラムが他者のものと酷似している場合や、不正な処理が含まれている場合は、採点対象外とする
ことがある。


=======

本課題では，学生証番号の下1桁に基づいて，A〜Eの5つのグループに分かれます。

学生証番号の下1桁	グループ
0, 5	グループA
1, 6	グループB
2, 7	グループC
3, 8	グループD
4, 9	グループE
各グループには，それぞれ異なるプロジェクト用データが割り当てられています。

以下に表示されているファイルのうち，自分のグループに対応するZIPファイルをダウンロードしてください。Moodleのアクセス制限により，自分のグループに対応するファイルのみダウンロードできます。

ダウンロードしたZIPファイルには，次の4ファイルが含まれています。

train.csv：モデルの学習に使用する訓練データ（グループごとに異なる）

test_sample.csv：提出プログラムの動作確認に使用するサンプルデータ（グループごとに異なる）

student_template.py：提出用Pythonプログラムのテンプレート（グループ共通）

development.ipynb：Google Colab上で提出プログラムを作成・確認するためのノートブック（グループ共通）

ZIPファイルを展開し，これらのファイルを使用して課題に取り組んでください。

自分の学生証番号に対応しないグループのデータを使用しないでください。表示されるZIPファイルが自分のグループと一致しない場合は，課題を開始する前に担当教員へ連絡してください。
