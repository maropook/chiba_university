# 第4週目 課題4 レポート

## 課題内容
DC命令に与えられる2値について、その積の下位16ビットを求め、DS命令で定義されるメモリ領域に格納するプログラムを作成せよ。
シミュレーション実験によって #005F × #007D を計算し、計算にかかる時間、実行する命令数、CPI値を示せ。

---

## アルゴリズム：足し算繰り返し法

データa（GR1）をデータb（GR2）回加算することで積を求める。

```
手順:
1. GR0を0で初期化する
2. GR1をGR0に加算する
3. GR2から1を減算する
4. GR2が0でなければ2へ移動する
5. 呼び出し元へ戻る
```

---

## CASL IIプログラム

```
MAIN1    START
         LD      GR1, DATA1   ; GR1 = #005F（被乗数）
         LD      GR2, DATA2   ; GR2 = #007D（乗数）
         CALL    MULT1        ; 乗算副プログラム呼び出し
         ST      GR0, ANS     ; 結果をメモリに格納
         RET
DATA1    DC      #005F        ; 被乗数
DATA2    DC      #007D        ; 乗数
ANS      DS      1            ; 結果格納領域
         END

MULT1    START
         PUSH    0, GR1       ; GR1をスタックに退避
         PUSH    0, GR2       ; GR2をスタックに退避
         LAD     GR0, 0       ; GR0 = 0（結果初期化）
LOOP     ADDA    GR0, GR1     ; GR0 += GR1
         SUBA    GR2, ONE     ; GR2--
         JNZ     LOOP         ; GR2≠0ならループ継続
         POP     GR2          ; GR2を復元
         POP     GR1          ; GR1を復元
         RET                  ; 呼び出し元へ戻る
ONE      DC      1
         END
```

======================

MAIN1    START
         LD      GR1, DATA1
         LD      GR2, DATA2
         CALL    MULT1 
         ST      GR0, ANS 
         RET
DATA1    DC      #005F  
DATA2    DC      #007D  
ANS      DS      1       
         END

MULT1    START
         PUSH    0, GR1   
         PUSH    0, GR2     
         LAD     GR0, 0      
LOOP     ADDA    GR0, GR1    
         SUBA    GR2, ONE   
         JNZ     LOOP      
         POP     GR2     
         POP     GR1        
         RET               
ONE      DC      1
         END

### 工夫した点
- CALL命令で副プログラムとして実装し、メインプログラムと分離した
- PUSH/POP命令でGR1,GR2の元の値を保護し、呼び出し元に影響を与えない設計にした
- LAD GR0,0 命令でGR0を確実に0初期化した

## シミュレーション結果

### 実験環境
- クロック周波数: 125MHz（周期8ns）
- タイムスケール: 1ns/1ps
- START立ち上がり: 122ns

### 計算結果
```
memory[11] = #2E63
```
検算: #005F × #007D = 95 × 125 = 11875 = #2E63 ✓

### 計算時間
```
DONE=1になった時刻: 28,980ns
計算時間 = 28,980ns - 122ns = 28,858ns
```

### 命令数

| 区間 | 命令 | 命令数 |
|------|------|--------|
| MAIN1 | LD, LD, CALL, ST, RET | 5 |
| MULT1 初期化 | PUSH, PUSH, LAD | 3 |
| ループ（125回） | ADDA + SUBA + JNZ | 375 |
| MULT1 後処理 | POP, POP, RET | 3 |
| **合計** | | **386** |

ループ回数が125回になる理由: DATA2 = #007D = 10進数125であり、SUBA命令で毎回1減算して0になるまでループするため。

### CPI

```
総クロック数 = 28,858ns ÷ 8ns = 3,607クロック
命令数 = 386命令
CPI = 3,607 ÷ 386 ≈ 9.35
```
