## 【4/16】ヒューマンインタフェースのモデル

### 実行と評価におけるへだたり

- **実行のへだたり**：それをどうやって動かすのか？ 何ができるか？
- **評価のへだたり**：何が起こったのか？ これは私が望んでいたことか？
- **デザイナーの役割**：二つのへだたりに橋を架ける手助けをする

ユーザがへだたりを乗り越えようとした際に、詳細化・プラン・実行の段階で困難を感じたり、失敗をしたりする。

```
物理的世界 ←─ 実行のへだたり ─→ [外界] ←─ 評価のへだたり ─→ 心理的世界
```

---

### 行為の7段階理論

心理的世界（ゴール）から物理的世界（外界）へ至る「実行の橋」と、外界から心理的世界へ戻る「評価の橋」の7つのプロセス。

```
          [心理的世界]
               ゴール
              /       \
     [実行の橋]       [評価の橋]
          /                 \
       プラン              比較（ゴールと結果の比較）
         |                    ^
       詳細化               解釈（知覚したものの解釈）
         |                    ^
       実行                 知覚（外界の状態の知覚）
              \       /
           [物理的世界・外界]
```

**行為の7段階理論と基本的なデザイン原則（問うべき基本的なチェックリスト）**

| 段階 | 問い |
|------|------|
| ゴール（ゴールの形成） | 何を達成したいか？ |
| プラン（行為のプラン） | 代替となるプラン、行為は何か？ |
| 詳細化（行為系列の詳細化） | 何ができるか？ |
| 実行（行為系列の実行） | それをどうやってやるのか？ |
| 知覚（外界の状態の知覚） | 何が起こったのか？ |
| 解釈（知覚したものの解釈） | それは何を意味するのか？ |
| 比較（ゴールと結果の比較） | それで良いか？ |

---

### 概念モデル

```
[デザイナー] ─── (デザインモデル) ───┐
                                     ▼
                              [システムイメージ]
                           (システム/ドキュメンテーション)
                                     ▲
[ユーザ]     ─── (ユーザモデル) ────┘
```

**デザインモデル**

- システムの設計者（デザイナー）がシステムに対して持つ概念モデル
- システムの機能：システムにどのような機能を持たせ、どのように使うか
- 機能を取り扱う上で、設計者が抱いている基本的なデザイン方針や具体的なデザイン案

**システムイメージ**

- 構築された物理的な構造（文書を含む）から知覚されるイメージ
- 入力に対するフィードバックやエラーメッセージ
- 操作手順書、マニュアル、オンラインヘルプ、インタフェースデザインそのもの
- デザイナーがユーザにデザインモデルを伝える**唯一の手段**

**ユーザモデル**

- ユーザがシステムに抱くメンタルモデル
- メンタルモデル：自分自身、他者、環境、その人が関わりを持つもの等に対して個人が持つモデル（人間は説明好きな存在）
- 製品やシステムイメージとのインタラクションによって形作られる
- ユーザが抱くシステムの構造や動作のイメージ。このモデルに基づき、ユーザはシステムの挙動を予測したり、結果に基づき修正したりする

### 冷蔵庫の例（Norman, 1986より）

冷蔵室と冷凍室の二つの区画があり、二つのコントロールスイッチが冷蔵室内にある。

**問題**：冷凍室が冷えすぎていて、冷蔵室は適温であるとしよう。冷蔵室の温度はそのままで冷凍室の温度を上げるためには、このコントロールスイッチをどう操作したらよいだろうか。

| 設定 | コントロール |
|------|------------|
| 停止（冷蔵・冷凍） | C |
| 冷蔵室 弱冷 | D |
| 標準 | P / 5 |
| 冷蔵室 強冷 | D / 6-7 |
| 冷蔵室 最強冷 | D / 8-9 |
| 冷凍室 強冷 | C / 7-8 と 4-10 |

※注意：両方とも設定すること。温度安定は24時間後。

```
[冷凍室]  A  B  C  D  E      [冷蔵室] 7  6  5  4  3  2  1
```

### デザインの7つの基礎的な原理

- **アフォーダンス**：望ましい行為を可能にするために適切なアフォーダンスがある。
- **シグニファイア**：効果的にシグニファイアを利用することによって、発見可能性を確かなものにし、フィードバックが理解可能なかたちで伝えられる。
- **制約**：物理的、論理的、意味的、文化的な制約を与える。これによって行為を導き、解釈のしやすさを助ける。
- **フィードバック**：行為の結果と製品やサービスの現在の状態についての完全かつ継続的な情報がある。行為が実行された後、新しい状態がどうなったかが分かりやすい。

---

### ◎（ギブソンによる）アフォーダンス

- 物理的なモノと主体（人など）の「関係性」
- モノの属性と、それとどのようにインタラクションできるか（どのように使うことができるか）を決定する主体の能力、との間の関係性
- 主体は動物でも人でも機械やロボットでもよい

**（例）椅子**
- 椅子は支えることをアフォードする → 座ることをアフォードする
- 一人で運ぶことができる椅子 → 持ち上げることをアフォードする
- ※力の弱い人が椅子を持ち上げられなかったとしたら、その人にとって、その椅子は持ち上げることをアフォードしない

→ アフォーダンスはモノの性質ではなく**関係性**である

---

### ◎ 知覚されるアフォーダンス

アフォーダンスは目に見えなくても存在する。

- 知覚される（目に見える）アフォーダンスは、モノを操作するときに強力な手がかりを提供する
- ドアに付けられた平らな板は「押すこと」をアフォードする
- ノブは「回すこと、押すこと、引くこと」をアフォードする
- 小さな細長い穴は「何かを挿入すること」をアフォードする
- ボールは「投げる、バウンドさせること」をアフォードする

知覚されたアフォーダンスは、どんな行動をとりうるかを表示や説明の必要なしに思い描く助けになる。

---

### ◎ アフォーダンスと反アフォーダンス

**透明なガラスの例**：
- 「透かして見ること」をアフォードする
- ほとんどの物理的なモノが通り抜けることを阻止 ＝ 反アフォーダンス

アフォーダンスや反アフォーダンスが有効に働くためには「知覚可能（発見可能）」であることが必要。

透明なガラスは反アフォーダンスの性質（通り抜けを阻止）を隠してしまう。
- → 鳥が窓を通り抜けて飛ぼうとしてガラスに衝突する
- → 人がガラスドアを走り抜けようとして怪我をする

---

### ◎ シグニファイア

アフォーダンスの存在を示す特性。

- アフォーダンスが知覚可能でなければ、その存在を示す何らかの手段が必要となる
- 実世界に置かれたサイン、ラベル、絵など（例：ドアに書かれた「押す」「引く」「出口」）
- 何をすべきか、あるいはどの方向に動けばよいのかの矢印や線図、その他の指示表示
- シグニファイアは「知覚されるアフォーダンス」も含む（例：ドアの取っ手やスイッチの物理的構造）

知覚されたアフォーダンスには真でないもの（間違いを誘うもの）もある：
- 実際にはそうでないのに押せるように見えるドア
- 入れないように見える入り口
- → 間違えさせるシグニファイア（偶発的／意図的）（例：1月公開の道路の入り口に立てられたゴム製ポール）

---

### ◎ 電子ディスプレイのグラフィカルデザイン

- ユーザが指で触るべきところを示すために丸を表示 → これはアフォーダンスではなく、**シグニファイア**
- 画面そのものは触る行為をアフォードしている（アフォーダンス）
- 丸は「どこを触ればよいか」を伝える（シグニファイア）

| 概念 | 役割 |
|------|------|
| アフォーダンス | どのような行為が可能かが決定される |
| シグニファイア | どこでその行為が行われるべきかが伝えられる |

---

### ◎ アフォーダンスとシグニファイアのまとめ

**アフォーダンス**：
- 主体（人間、動物、機械）が何かとどうインタラクションできるかについて、実世界での可能性を示す
- 知覚可能なものとそうでないものがある
- アフォーダンスが容易に発見できるのは知覚可能な場合のみ（＝知覚されたアフォーダンス）
- 知覚されたアフォーダンスはシグニファイアとして働くことが多いが、あいまいな場合もある

**シグニファイア**：
- どんな行為が可能か、それがどう行われるべきかを示す
- シグニファイアは知覚されるものでなければならない（そうでないときはうまく機能しない）
- 人々が可能な行為を決定するのを助ける

---

### 制約とは

| 種類 | 例 | 説明 |
|------|-----|------|
| 物理的制約 | USBケーブル、鍵、電池 | 物理的に可能な操作の幅を制限する |
| 意味的制約 | オートバイの風よけは人間の前に配置 | 可能な行為の集合を状況の意味に基づいて制限する |
| 文化的制約 | エレベータに乗る向き | 許容される行為の集合を文化的な慣習に基づき制限する |
| 論理的制約 | 2口ガスコンロのつまみと火口の配置 | 対象間の論理的関係性によって対応付けを制限する |

---

### フィードバック

フィードバック：行為の結果を伝える。

**悪い例**：エレベータや横断歩道のボタン（海外の、押しても反応が分からないもの）

**原則**：
- フィードバックはすばやくなくてはならない
- フィードバックは豊かな情報を持たなければならない
- フィードバックは多すぎてもいけない（適切に出されなければならない）

**メディアごとの特徴**：
- ボタン：押した時の反力、ボタンが光る
- ランプ：見えないと気付かない
- ビープ音：どの機械が出しているか分かりにくい

---

### まとめ：発見可能性

どのような行為が行えるのか、機器の今の状態がどうなっているのかが判断できること。

- 対応づけ ＝ 概念モデル
- アフォーダンス
- シグニファイア
- 制約
- フィードバック

## 【6/18】ヒューマンエラーと対策

### ◎ ヒューマンエラー

エラー → 「適切な」振る舞いからの逸脱  
※「適切な」：不明であったり、事後にのみ判明することも多い

```
                    ┌── スリップ（行為ベース）
         ┌── エラー ┤
         │          └── 記憶ラプス
人間エラー─┤
         │          ┌── ルールベース
         └── ミステーク ┤
                    ├── 知識ベース
                    └── 記憶ラプス
```

---

### ミステークとスリップ

**ミステーク（Mistake）**：計画の失敗 → ユーザが気づきにくい
- 目的や計画（判断）そのものが間違っていたため、その間違った計画通りに行動してしまうエラー
- ルールベース：誤った行為を選択してしまう
- 知識ベース：誤った知識や不完全な知識で間違える
- 記憶ラプス：ゴール、プラン、比較の段階で忘却

**スリップ（Slip）**：実行の失敗 ＝ ユーザが気づきやすい
- 目的や計画は正しかったのに実際の行動（操作）を間違えてしまうエラー
- 実際の行為は意図した行為とは異なる
- 行為ベース：間違った行為が行われる（例：コーヒーにミルクを注いだ後、カップを冷蔵庫にしまう）
- 記憶ラプス：記憶が失われ、意図した行為が行われないか、結果が評価されない

---

### スリップの型（1/2）

**乗っ取り型スリップ**
- 習熟度の低い操作に代わって習熟度の高い操作が遂行されてしまう
- 例：あまり集中せずにある曲を演奏しているときに、気がつくと良く知っている曲を演奏していた

**記述類似性スリップ**
- 操作系列の内的な記述が不十分なために、類似の操作が遂行されてしまう
- 例：オレンジジュースをコップに注ごうとしてコーヒーカップに注いでしまった（隣に並んでいた）

---

### スリップの型（2/2）

**モードエラースリップ** ＝ デザインの問題
- 別のモードでは正しいが現在のモードでは不適切である操作を遂行してしまう
- 例：ひらがなをアルファベットに戻そうと [Ctrl]+[P] を押したら印刷コマンドが起動した（ワード使用時はIMEのショートカットよりワードのショートカットが優先）

**記憶ラプススリップ**
- その行為を何のためにするつもりだったか思い出せないまま継続している状態
- 例：実験室に行く用事があったので、歩き出したのだが、ふと、なぜ実験室に向かっているのか思い出せなくなった

---

### ◎ エラーと行為の7段階理論

```
         [心理的世界]
              ゴール  ┐
             プラン   ├── ミステーク（計画段階のエラー）
            詳細化    ┘
     実行の橋  |        評価の橋
              実行  ─┐
              知覚    ├── スリップ（実行段階のエラー）
              解釈    │
              比較  ──┘
         [物理的世界・外界]

★記憶ラプスで行為のサイクルが止まるため、望んだ行為が完了されない
```
