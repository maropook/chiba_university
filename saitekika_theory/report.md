## 1. 実装の概要

### 1.1 関数インタフェース

```python
simplex(c, A, b, sense="max", verbose=False) -> dict
```

| 返り値キー | 内容 |
|---|---|
| `status` | `"optimal"` / `"unbounded"` |
| `x` | 主問題の最適解ベクトル |
| `objective` | 最適値 |
| `y` | 双対最適解（最終タブローの z 行から読み取り）|
| `num_iterations` | ピボット回数 |
| `basis` | 最終基底変数の添字リスト |
| `slack` | 各制約のスラック値 |

### 1.2 アルゴリズムの概要

タブロー単体法を以下の 3 ステップで繰り返す。

```
ステップ1（入る変数の選択）
  z 行で最も負の係数を持つ列を選ぶ（Dantzig 規則）。
  全係数 ≥ 0 ならば最適として停止。

ステップ2（出る変数の選択）
  各行で RHS / ピボット列係数 を計算し（係数 > 0 の行のみ）、
  最小比の行を出る変数とする（最小比検定）。
  全比率が ∞ ならば非有界として停止。

ステップ3（ピボット操作）
  ピボット行をピボット要素で割り、他のすべての行からピボット行の倍数を引く。
```

**双対解の読み取り**：最終タブローの z 行において、スラック変数 sᵢ の位置にある係数が双対変数 yᵢ* となる。

### 3.3 考察：被約費用と相補性条件

**被約費用（reduced cost）の意味**：

被約費用 c̄ⱼ = cⱼ − A_j^T y* は、変数 xⱼ を 1 単位増やしたときの「純粋な利益の増分」を表す。双対解 y* によって制約の機会費用を差し引いた後の利益改善率である。

- c̄ⱼ < 0（最終タブローの z 行が負）→ xⱼ を増やすと利益が上がる → まだ改善できる（非最適）
- c̄ⱼ ≥ 0（最終タブローの z 行が非負）→ xⱼ を増やしても利益は増えない → 最適条件

**相補性条件（Complementary Slackness）の意味**：

主問題と双対問題の最適解が対応するとき、以下の条件が成立する。

- **主側**：xⱼ > 0 ならば c̄ⱼ = 0（基底変数の被約費用はゼロ）
- **双対側**：sᵢ > 0（制約に余裕あり）ならば yᵢ* = 0（その制約の影の価格はゼロ）

言い換えると、「使い切っていない資源には価値がつかない」「生産している製品の利益余地はゼロ」ということを意味する。

**本問題での確認**（プログラム出力より）：

| 変数 | 値 | 被約費用 c̄ | xⱼ × c̄ⱼ | 意味 |
|---|---:|---:|---:|---|
| x₁ | 2 | 0 | 0 | 基底変数（生産中）→ c̄=0 ✓ |
| x₂ | 0 | −3 | 0 | 非基底（不生産）→ x=0 で積=0 ✓ |
| x₃ | 1 | 0 | 0 | 基底変数（生産中）→ c̄=0 ✓ |

- **x₂=0, c̄₂=−3**：x₂ は非基底。最終タブローの z 行で x₂ 列の係数が +3 > 0 であり、x₂ を増やすと z が減少するため最適解では生産しない。x₂ × c̄₂ = 0 で相補性条件成立。
- **s₂=1 > 0, y₂*=0**：制約2に余裕（スラック1）があるため、双対変数 y₂*=0。余っている資源に影の価格はつかない。双対側の相補性条件も成立。

## 5. 設計の工夫・考察

### 5.1 数値安定性

停止条件・最小比検定の正値判定に閾値 ε = 10⁻⁹ を用いた。

```python
EPS = 1e-9
if z_row[entering_col] >= -EPS:   # 停止条件
if coef > EPS:                    # 比率計算の対象（正の係数のみ）
```

### 5.2 任意サイズへの対応

タブローのサイズを `(m+1) × (n+m+1)` として動的に生成するため、変数・制約の数に関わらず同一コードで対応できる。課題3の 8変数×6制約（頂点候補 ₁₄C₆ = 3003）も7回のピボットで最適解に到達した。

### 5.3 双対解の抽出

最終タブローの z 行においてスラック変数 sᵢ に対応する列の係数を読み取ることで、双対最適解 y* を自動的に取得している。

### 5.4 verbose モード

`verbose=True` を指定することで、各反復のタブローが変数名付きで出力される。課題1・2の手計算との照合に活用した。
